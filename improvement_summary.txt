================================================================================
IMPROVEMENT SUMMARY: STATE-LEVEL TRANSFORMER FORECASTING & EVALUATION
================================================================================

1. WHAT THE IMPROVEMENT DID
--------------------------------------------------------------------------------
This improvement transitioned the influenza-like illness (ILI) forecasting pipeline 
from a flawed global dataset splitting strategy to a robust, per-state split training 
and evaluation pipeline. It also introduced a comprehensive evaluation comparing 
the Deep Transformer model against ARIMA(3,0,3) and Persistence baselines.

Specifically, the following were implemented and executed:
* Corrected Data Splitting: Modified the data preparation flow. Instead of concatenating 
  all states' historical data together and applying a global temporal split at 80% 
  (which resulted in alphabetical end-of-list states representing the entire test set), 
  the dataset now splits each state's individual timeline (YEAR/WEEK) at 80% for training 
  and 20% for testing before concatenating them. This ensures every state is represented 
  in train, validation, and test datasets.
* Deep Transformer Model: Trained and saved the best weights of a Deep Transformer model 
  with sinusoidal positional encoding, 4 encoder layers, and 4 decoder layers on the 
  corrected state-level dataset.
* Baselines Implementation: Implemented an autoregressive ARIMA(3,0,3) model and a 
  Persistence model (using the last observed value) as comparison benchmarks.
* Evaluation Metrics: Computed RMSE, MAE, and Pearson Correlation Coefficients for all 
  models across each of the 4 forecast horizons (Weeks +1 to +4) and overall.
* Visualization: Generated three comparative plots:
  - state_transformer_training_curves.png: Training and validation loss curves.
  - state_model_comparison.png: Bar chart comparisons of overall RMSE, MAE, and Pearson Correlation.
  - state_model_predictions_examples.png: Comparative prediction curves vs. ground truth 
    for four example states (California, Texas, New York, Florida).


2. THE THOUGHT PROCESS & METHODOLOGY
--------------------------------------------------------------------------------
* Data Splits and Temporal Consistency:
  In time-series forecasting, especially with spatial components (states), global splitting 
  by simple array index slicing can introduce severe selection bias. Because states are 
  listed alphabetically in the CSV, a global 80/20 split puts the first ~43 states' timelines 
  fully into the training set and the last ~11 states (alphabetically) fully into the test set. 
  By partitioning locally within each state's individual time-series before concatenating, 
  we allow the model to learn spatial features invariant to region names and evaluate 
  generalization performance fairly across all states.
* Debugging Shape Errors:
  When running the evaluations, a ValueError was encountered due to dimension squeezing on 
  a 2D numpy array representing the test window (shape (N, 10)). The code was adjusted to 
  properly format inputs to 3D tensors (shape [N, 10, 1]) for PyTorch models while keeping 
  appropriate shapes for ARIMA and Persistence baselines.
* ARIMA Optimization & Fallback:
  ARIMA models were fitted using statsmodels on each individual 10-week window to predict 
  4 weeks into the future. Since fitting ARIMA dynamically on thousands of short, noisy 
  windows can lead to convergence failures or non-stationary initial parameters, a robust 
  fallback mechanism was added to repeat the last value if optimization failed, preventing 
  script crashes.


3. FINAL EVALUATION METRICS COMPARISON
================================================================================
The following metrics were calculated over the complete test set consisting of 
4,332 test windows across 54 states/territories:

Model / Horizon                 RMSE        MAE    Pearson Corr
================================================================================
  Week+1 Persistence           0.7216     0.3984          0.9482
  Week+2 Persistence           1.0458     0.5783          0.8902
  Week+3 Persistence           1.3315     0.7387          0.8212
  Week+4 Persistence           1.5946     0.8991          0.7424
--------------------------------------------------------------------------------
  Overall Persistence          1.2176     0.6536          0.8510
================================================================================
  Week+1 Arima                 0.9679     0.5876          0.9078
  Week+2 Arima                 1.4813     0.9001          0.7872
  Week+3 Arima                 1.9539     1.1998          0.6429
  Week+4 Arima                 2.3134     1.4480          0.5107
--------------------------------------------------------------------------------
  Overall Arima                1.7536     1.0339          0.7080
================================================================================
  Week+1 Transformer           0.6910     0.3790          0.9507
  Week+2 Transformer           0.9574     0.5230          0.9016
  Week+3 Transformer           1.1594     0.6313          0.8501
  Week+4 Transformer           1.3234     0.7302          0.7969
--------------------------------------------------------------------------------
  Overall Transformer          1.0595     0.5659          0.8773
================================================================================

Key Findings:
* Deep Transformer Dominance: The Deep Transformer model achieved the best performance 
  overall, with an overall RMSE of 1.0595 compared to Persistence (1.2176) and ARIMA (1.7536).
* Long-term Robustness: The gap between the Transformer and Persistence grows wider at 
  further horizons. At Week+1, the difference in RMSE is small (0.6910 vs. 0.7216), but by 
  Week+4, the Transformer retains an RMSE of 1.3234, whereas Persistence degrades to 1.5946.
* ARIMA Limitations: ARIMA performed the poorest here. Fitting short 10-week timelines 
  sequentially is highly sensitive to noise, and a linear statistical model fails to 
  efficiently capture nonlinear epidemical spike dynamics.
